XML概念

XML（可扩展性标记语言）是一种平台独立性语言，用于描述数据和跨平台数据交换。所谓XML解析，是指将XML报文映射成响应的JAVA类对象，便于业务逻辑中使用。JAVA解析XML有几种方式，大体上讲只有SAX（Simple API for XML）和DOM（Document Object Model）两种方式，这两种方式解析XML比较难用，一些软件组织在这两种方式的基础上做了一些封装，提供了相对友好的解析方式（如DOM4J，JDOM等）。

DOM和SAX

概念

DOM是XML标准提出者w3c组织提出的一种规范，DOM方式概括的讲是将整个XML文件对应的建立一个树状结构的对象放入内存，然后以对象的形式解析。SAX方式类似与流媒体，基于事件触发，比如重头到尾浏览整个XML文件遇到一些标签，会促发一些解析的方法。

对比

对比而言，DOM方式不适合解析很大的XML，在XML文件很大时会导致内存溢出，但是这种方式解析XML由于将整个对象放入内存在对其修改时很方便；而SAX方式基于流不一次读入所有的，可以用很少的内存资源，SAX有个缺点是只能按顺序进行，解析过程中不方便往回或者跳跃的去处理前面的内容，适合读取不适合修改XML内容。

JDOM，DOM4J和JAXB

DOM4J

据测试，DOM4J的性能是最好的，hibernate等框架也是使用DOM4J来解析XML文档。DOM4J中有一个对象Document，可以说它是文件或者String的一种抽象，可以方便的对其添加或者删除节点，然后再保存到文件或者字符串中。DOM4J支持xPath，可以方便快速的读取XML的内容。

DOM4J一般只需要使用包dom4j-1.6.1.jar，使用xPath时需要包jaxen-1.1.6.jar。

JDOM

JDOM与DOM4J类似，性能稍逊，语法有别，需要做类型转换，核心jar包jdom-2.0.6.jar。

JAXB

SUN提出，JDK自带，原理是根据DTD生成java源文件，然后再解析，自动做类型转换。处理时与java源文件绑定，如果想增删节点，则必须相应的修改java源文件，不便于做增删操作。
