长连接与短连接

首先这里的连接是指TCP的连接，短连接指每次HTTP请求结束后马上断开连接，HTTP协议最迟这么设计的初衷是为了减少服务器的压力，但是现在的网页经常一个HTML页面过来后里面又有很多的URL，如果采用短连接的方式反复连接也会很耗资源。所有有了长连接（Keep-Alive），每次不马上释放连接。

get与post

HTTP协议是我们平时上网，浏览器经常使用的协议，其中HTTP请求的提交里有GET和POST两种方式，在java Web开发中经常遇到。
GET用于请求服务器资源，POST用于更新服务器的某些资源。

GET方式

当以get方式提交时，观察浏览器页面的地址栏中URL为

http://localhost:8080/test/HeroLogin?username=berry&password=123456

抓包后，可以看到报文

GET /test/HeroLogin?username=berry&password=123456 HTTP/1.1
Host: localhost:8080
User-Agent: Mozilla/5.0 (X11; Ubuntu; Linux i686; rv:39.0) Gecko/20100101 Firefox/39.0
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,/;q=0.8
Accept-Language: zh-CN,zh;q=0.8,en-US;q=0.5,en;q=0.3
Accept-Encoding: gzip, deflate
Referer: http://localhost:8080/
Connection: keep-alive
HTTP/1.1 200 OK
Server: Apache-Coyote/1.1
Content-Length: 0
Date: Sat, 25 Jul 2015 05:13:13 GMT

可以看到GET方式请求信息是通过URL提交的。

POST方式

观察到以post方式提交，浏览器地址栏URL中没有页面的信息。

http://localhost:8080/test/HeroLogin

抓包后可以看到

POST /test/HeroLogin HTTP/1.1
Host: localhost:8080
User-Agent: Mozilla/5.0 (X11; Ubuntu; Linux i686; rv:39.0) Gecko/20100101 Firefox/39.0

Accept-Language: zh-CN,zh;q=0.8,en-US;q=0.5,en;q=0.3
Accept-Encoding: gzip, deflate
Referer: http://localhost:8080/
Connection: keep-alive
Content-Type: application/x-www-form-urlencoded
