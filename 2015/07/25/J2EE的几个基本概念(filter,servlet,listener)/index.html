基本概念

filter

过滤器,使用责任链模式实现.对所有的请求进行过滤,多个过滤器之间相互独立.

servlet

servlet是一种运行服务器端的java应用程序,并且可以动态的生成web页面，工作在客户端请求与服务器响应的中间层。

listener

监听器,使用观察者模式实现.主要用来监听。当application，session，request三个对象创建消亡或者往其中添加修改删除属性时自动执行代码

实现

filter

实现接口javax.servlet.Filter
web.xml配置

  1. 多个filter之间相互独立,初始化化顺序按照web.xml中<filter>配置顺序.过滤请求按照web.xml中<filter-mapping>顺序,如果所有的filter都通过,交给servlet处理请求.
  2. 过滤器的dofilter方法中chain.doFilter(request, response);将请求传递下去,如果没有该代码,所有的请求都将在该过滤器中被终止.

servlet

Http协议的servlet实现,可以继承javax.servlet.http.HttpServlet类,重写doGet和doPost方法.请求和响应被封装在实现HttpServletRequest和HttpServletResponse接口的类对象中.

  1. 一般在doPost方法中实现具体业务处理的代码,处理完成给出响应.
  2. servlet一般用来写纯http后台接口,不涉及网页页面.

web.xml配置

listener

servlet2.4规范中提供了8个listener接口，可以将其分为三类:
1.与servletContext有关的listner接口,包括：ServletContextListener、ServletContextAttributeListener.
2.与HttpSession有关的Listner接口,包括：HttpSessionListner、HttpSessionAttributeListener、HttpSessionBindingListener、HttpSessionActivationListener.
3.与ServletRequest有关的Listener接口，包括：ServletRequestListner、ServletRequestAttributeListener.

生命周期

filter

filter类在web服务器启动初始化时即被加载,实例化时会执行init方法.在web服务器关闭时,会执行destroy方法.

servlet

如果已配置自动装入选项，则在启动服务器时自动装入;否则在匹配的请求进来时才会加载.实例化时执行init方法.

总体,初始化顺序:context-param > Listener > Filter > Servlet
