序列化

首先，序列化是对对象而言的。将一个对象编码成一个字节流，称为该对象序列化；相反的处理过程称为反序列化。序列化的类可以通过网络传输或者保存在磁盘上，供以后反序列化。

用途：

  1. RMI用来传输远程方法的调用参数和返回值。
  2. GUI用来保存JavaBean的状态。
  3. Throwable实现序列化接口以使得RMI可以返回异常，Component实现序列化使得组件可以被存储和读出，HttpServlet实现序列化使得它可以被缓存。

实现:

  1. 要序列化的对象的类需实现Serializable接口。
  2. 以深克隆为例，创建ObjectOutputStream实例，ObjectOutputStream构造器入参为要把对象写入的流（OutputStream子类），比如内存ByteArrayOutputStream，磁盘FileOutputStream，或者网络。序列化类，调用ObjectOutputStream对象的writeObject()方法即可。
  3. 序列化读出来的对象要做强制转换。也就是RMI调用时，客户端/服务端都需要知道对象类的类文件。

注意

  1. 对于GUI等程序要谨慎使用序列化，因为如果一个类的成员发生变化会导致，以前序列化在磁盘上的对象无法再恢复。而RMI对于该问题并不敏感，因为通过网络传输序列化与反序列化的时间周期短。
  2. 需要明确指定序列化ID，如果不指定运行时系统会根据对象内容自动生成，如果类的结构改变，每次生成的UID会不同，这回引起InvalidClassException异常（序列化ID用来标示类的版本，如果反序列化时ID不同则类肯定将不能被反序列化，直接抛异常）。
  3. 对象序列化时会自动将该对象内部涉及到的所有对象序列化（要求它们必须都实现Serializable接口）。
  4. 关键字transient可以过滤不想序列化的字段，反序列化时该字段为空值，该关键字只对Serializable接口对象有效。（安全考虑，即便把这些字段设为私有的也要防止利用反射机制读出）
  5. 实现Externalizable接口覆写方法writeExternal和readExternal可以定制序列化。实现该接口，除了反序列化时会自动初始化成员变量和调用默认构造器不会做任何事，需要自己手动定制需要序列化和反序列化的字段。
  6. 另一种定制方法：在Serializable接口对象中添加两个私有方法writeObject和readObject，在序列化时ObjectOutputStream会通过反射自动调用取代默认生成的方法。在定制方法中，可以调用ObjectOutputStream类的defaultReadObject和defaultWriteObject方法完成原来的默认序列化过程。
