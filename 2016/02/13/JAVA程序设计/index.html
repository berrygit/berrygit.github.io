基本运算

移位

<<向左移位，空位补零；
>>向右移位，移位时符号位不动。正数空位补零，移出的1直接舍去；负数空位也补零，移出的1舍去后再加在原数字的最低位上；
>>>无符号向右移位，对于负数，首先符号位不变，向右移动一个单位，空位补零，之后将所有位取反，将负数变为正数，之后按照正数的移位方式进行。

访问控制

类和接口只有public和private两种可访问级别，如果没有public修饰，就是private的。私有意味着只能被同一个包下的类访问。

成员（域，方法，嵌套类，嵌套接口）有四种访问级别：
私有，声明该成员的顶层类内部才能访问。
包私有，同一个包下。
受保护，同一个包下或者它的子类。
公有，任何地方。

子类的成员的可访问性不能低于父类，保证里氏替换原则。

final

修饰类，类是不可子类化的。

类

构造器

  1. 类中如果没写构造器，编译器会自动提供一个默认构造器；类中如果提供了构造器，编译器将不会提供任何构造器。
  2. 私有构造器可以使类及其子类无法被实例化，某些只提供静态成员的类需要这么做来防止无意义的实例化。

初始化

  1. 局部变量声明时就必须显式初始化；类变量声明时会自动初始化为默认值。
  2. 类的静态成员（类静态变量，静态初始化块）只初始化一次，当类被第一次new（其实构造器就是静态方法）或者类的静态变量和静态方法第一次被触及使用之前类加载器就会把类的静态成员全部初始化。
  3. 静态块只初始化一次，非静态块每次创建对象时都执行。
  4. 类静态成员先于一般成员；类变量先于初始化块；初始化块先于构造器。
  5. 先初始化父类然后子类，对于默认构造器，编译器会自动添加调用父类构造器的代码；如果不用默认构造器初始化，编译器会强制提醒显式调用父类构造器，并且是子类构造器内第一行代码，否则编译器会报错提醒。

Object类

Object类下除final方法外，其它方法都是应该按照需要被覆写的。具体有：

  1. equals，原始实现是通过==判断两对象是否指向同一引用，一般需要判断对象的值时需要覆写，如Map和Set中的元素是唯一的，插入或者查找时需要根据equals方法判断。存在继承关系的类无法满足传递性和对称性，需要使用组合替代继承。
  2. hashcode，集合中散列函数根据hashcode去放入和查找桶号，因而要求对于equals返回相等的对象它们的hashcode也必须相等，否则查找时找不到对象（注意计算hashcode时不要使用equals方法为判断的字段）；对于equals返回不等的对象尽量使它们的hashcode不等。由于计算hashcode的代价较高，可以在对象中缓存hashcode的值。
  3. toString，始终复写该方法。
  4. clone,见克隆。

克隆

实现

重载Object类的clone方法；调用super.clone()；同时类实现Cloneable接口。客户端程序调用clone方法时，JVM会自动将类中的基本类型及引用类型成员变量的值克隆。

深克隆与浅克隆

上述克隆只会克隆基本类型和引用类型的引用，称为浅克隆；对于引用类型需要使用序列化和反序列化的方式进行深克隆。深克隆要求对象内所涉及的对象全部要是可序列化的。

特例

  1. 数组的克隆，返回原类型。
  2. 不能克隆final域。
  3. 除非数组或只有基本类型域的copy，尽可能得不去调用clone方法，用静态工厂去实现。

标记接口

Serializable，Cloneable
不提供任何方法，只是一个标识，供以后识别方便。

继承

java使用接口来实现多重继承，一个类只能继承一个类，而接口可以同时继承多个接口。

使一个类不可子类化有两种方式：使用final修饰；类所有的构造器用private修饰。
